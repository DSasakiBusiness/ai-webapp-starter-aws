# Secure Release Pipeline

## この skill を使う場面

- CI/CD パイプラインにセキュリティチェックを組み込む場合
- リリース前のセキュリティゲートを設定する場合

## 入力前提

- GitHub Actions CI/CD が設定済み
- テストスイートが実行可能

## 実行手順

### Step 1: CI パイプラインのセキュリティチェック追加

1. 依存パッケージの脆弱性スキャン（`npm audit`）
2. 静的解析（ESLint セキュリティルール）
3. シークレットの混入チェック（git-secrets / TruffleHog）
4. Docker イメージの脆弱性スキャン（Trivy）

### Step 2: リリースゲート定義

リリース前に以下が全て合格していることを確認:
- 全テスト（unit / integration / E2E）合格
- セキュリティスキャン合格（Critical / High がゼロ）
- コードレビュー承認済み
- security-reviewer の承認

### Step 3: ロールバック手順の確認

- 前バージョンへの切り戻し手順を文書化
- DB マイグレーションのロールバック手順を確認

## 判断ルール

- Critical 脆弱性 → リリースブロック
- High 脆弱性 → security-reviewer の判断を仰ぐ
- Medium 以下 → 記録して次スプリントで対応

## 出力形式

GitHub Actions ワークフローファイルとリリースチェックリスト。

## 注意点

- セキュリティスキャンの誤検知は除外ルールで管理する
- シークレットの混入チェックは全コミットに適用する

## 失敗時の扱い

- スキャンツールがエラー: 手動チェックリストに切り替える
- 誤検知が多い: 除外ルールを調整し、ノイズを削減する

# Generate UI Spec

## この skill を使う場面

- 新しい画面を設計する場合
- 既存画面の改修仕様を定義する場合

## 入力前提

- product-manager の要件定義
- write-api-contract の API 契約

## 実行手順

### Step 1: 画面一覧

対象画面のルート、コンポーネント構成、データフローを定義する。

### Step 2: 画面ごとの仕様

各画面について以下を定義:
- URL パス
- コンポーネント階層
- 表示データ（API からの取得）
- ユーザーインタラクション
- 3 状態（ローディング、エラー、空）
- レスポンシブ対応

### Step 3: コンポーネント仕様

再利用可能なコンポーネントの Props、状態、バリアント  を定義する。

### Step 4: frontend-developer に共有

仕様書を渡し、implement-nextjs-ui skill で実装を開始する。

## 判断ルール

- 3 画面以上で使われる → 共通コンポーネント化
- AI 機能画面 → ストリーミング表示を前提とする

## 出力形式

UI 仕様書（画面一覧、コンポーネント仕様、状態遷移図）。

## 注意点

- 全画面にローディング・エラー・空状態を定義する
- data-testid の命名規則を含める

## 失敗時の扱い

- デザインが未確定: ワイヤーフレームレベルで仕様化し、後でビジュアルを追加

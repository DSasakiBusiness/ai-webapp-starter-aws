# Review Release Readiness

## この skill を使う場面

- ステージングから本番にデプロイする前
- 大きな機能リリースの前

## 入力前提

- 全テストの実行結果
- セキュリティレビュー結果
- AI 品質評価結果（AI 機能がある場合）

## 実行手順

### Step 1: チェックリスト確認

- [ ] 全ユニットテストが合格
- [ ] 全統合テストが合格
- [ ] 全 E2E テストが合格
- [ ] セキュリティスキャンで Critical/High がゼロ
- [ ] コードレビューが承認済み
- [ ] AI 評価スコアが基準以上
- [ ] DB マイグレーションが安全（破壊的変更なし）
- [ ] 環境変数の差分が `.env.example` に反映済み
- [ ] ドキュメントが更新済み
- [ ] ロールバック手順が確認済み

### Step 2: 合否判定

全チェック項目が合格 → リリース可能
1 つでも不合格 → 不合格項目の対応を先に行う

### Step 3: リリースノート作成

変更内容、既知の問題、破壊的変更（あれば）をまとめる。

## 判断ルール

- チェックリスト全合格 → リリース承認
- セキュリティ不合格 → security-reviewer に確認
- テスト不合格 → tdd-coach / e2e-tester に確認

## 出力形式

リリース判定結果（合否、不合格項目、リリースノート）。

## 注意点

- 「あとで直す」でリリースしない
- ロールバック手順は必ず事前に確認する

## 失敗時の扱い

- 判定に迷う場合: solution-architect と product-manager の合意を得る

# E2E テスト方針

## 概要

E2E テストはユーザーフロー全体をブラウザ上で検証する。Playwright を使用し、主要な動線が壊れていないことを保証する。

## 対象フロー

1. **認証**: ログイン / ログアウト / 認証エラー
2. **CRUD**: 一覧 / 作成 / 詳細 / 更新 / 削除
3. **AI 機能**: 入力 → 生成中表示 → 結果表示 → エラー → 再試行
4. **権限制御**: 未認証アクセス拒否 / 権限不足の操作拒否
5. **エラー表示**: 404 / 500 / ネットワークエラー

## テストフレームワーク

- Playwright

## 規約

### セレクタ
- `data-testid` ベースのセレクタを使用する
- CSS クラスやテキスト内容でのセレクタを避ける

### 認証
- `storageState` を使い、テストごとにログインしない

### テストデータ
- `beforeAll` でシード、`afterAll` でクリーンアップ
- テスト間でデータを共有しない

### 非同期操作
- `waitForSelector` / `waitForURL` / `waitForResponse` を使う
- `setTimeout` / 固定 sleep を使わない

## 実行方法

```bash
# サービス起動
make up

# E2E テスト実行
make test-e2e

# UI モード
make test-e2e-ui
```

## AI 機能の E2E テスト

- レスポンス構造と表示の正常性のみ検証する
- 文言の完全一致を避ける
- ストリーミング表示が正常に動作することを確認する

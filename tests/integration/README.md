# 統合テスト方針

## 概要

統合テストはモジュール間の連携を検証する。実際の DB（テスト用）を使い、API の入出力契約を検証する。

## 対象

- API エンドポイントの入出力
- DB アクセスを含むサービスレイヤー
- 認証・認可フロー
- AI サービスと API レイヤーの連携

## テストフレームワーク

- Jest + @nestjs/testing + PrismaClient

## 規約

### ファイル配置
- `[name].integration.ts`

### DB 管理
- テスト用 DB を使用する（開発用 DB とは分離）
- 各テストスイートの前に DB をクリーンアップする
- テストデータは `beforeAll` でセットアップする

### 実行方法

```bash
# Docker 経由（推奨）
make test-integration

# ローカル
npm run test:integration
```

## AI 機能の統合テスト

- モック LLM を使用するか、実 API の構造のみ検証する
- エンドツーエンドのデータフロー（リクエスト → AI → レスポンス）を確認する
- エラーレスポンスのフォーマットを検証する
